<no_lore>
The Requester is a robot-fed stock station. You set ghost templates for the items you want and how many; delivery robots linked to a station read those requests and bring matching items from the network until each request is filled.
</no_lore>
<chapter name="Information"/>
The <bold>request slots</bold> define what the station wants: set a ghost item in each slot and a requested count. These are templates only — nothing is consumed by placing them. Delivery robots that service this area read the requests and fetch matching items.
Delivered items land in the <bold>supplied items</bold> slots, one per request. A request counts as fulfilled once the stored amount reaches the requested count, so a robot fleet keeps a buffer of parts stocked automatically.
This is a robotics block — it needs robots and docking stations to do anything. See <link inline="buildcraft:concept/robots_and_zones"/> for the wider system.
